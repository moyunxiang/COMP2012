# COMP2012 

## 1. Review: Pointer, Reference, const-ness

### 1.1 Variable / Reference / Pointer

- **variable**: 有名字的内存存储，有地址
- **reference variable** (`T&`): 某个变量的 alias（别名）
  - 必须初始化，绑定后不可改变
  - 对 reference 赋值 → 改的是被引用对象的值，不是改绑定
- **pointer variable** (`T*`): 存的是地址
  - 可以为 `nullptr`
  - 可以在不同时刻指向不同对象
  - 需要 `*` / `->` 解引用

### 1.2 C++11 新特性

- `{}` 统一初始化（uniform initialization）
  - 比 `=` 更严格：**不允许 narrowing conversion**（如 `int x{6.4};` 报错）
  - 更通用：数组、aggregate struct、class 都可以用
- `nullptr` 替代 `NULL`
- `constexpr`: 编译期常量表达式
- range-based `for`:
  ```cpp
  for (int k : {0,1,2,3,4}) cout << k*k;
  for (int& x : range) x *= 2; // 可以修改
  ```
- `enum class`: 强类型枚举
- `override` / `final`
- rvalue reference & move semantics（见第 8 章）
- lambda（见第 5 章）
- delegating constructor / in-class member initializer

### 1.3 const 详解

- `const` 变量：值不可修改
- **const member function**: 函数不会修改对象状态
  - `const` 对象 **只能调用** `const` 成员函数
  - 非 `const` 对象两种都能调

- **const 与指针**的口诀：
  - `const` 在 `*` **左边** → 指向的对象是 const（不能通过指针改值）
  - `const` 在 `*` **右边** → 指针本身是 const（不能改指向）
  ```cpp
  const char* pcc;        // 指向 const char
  char *const cpc = &c;   // const pointer to char
  const char *const cpcc; // both const
  ```

- **const reference**: `const T&`
  - 不能通过它修改对象
  - 可以接受 const 和非 const 参数，也可以接受 literal

- 总结表：

  | 调用对象 | const 函数 | 非 const 函数 |
  |---------|-----------|-------------|
  | const 对象 | ✔ | ✘ |
  | 非 const 对象 | ✔ | ✔ |

  | 实参类型 | const ref 形参 | 非 const ref 形参 |
  |---------|-------------|---------------|
  | literal | ✔ | ✘ |
  | const 对象 | ✔ | ✘ |
  | 非 const 对象 | ✔ | ✔ |

### 1.4 Class 基本知识

- `struct` 默认 `public`；`class` 默认 `private`
- C++ 使用 **name equivalence**（不是 structure equivalence）
- **forward declaration**: `class Cell;` 允许使用指针/引用但不能访问成员
- **this pointer**: 每个非 static member function 隐含一个 `this` 指针
  - `*this` 就是对象本身

## 2. Constructor & Destructor

### 2.1 Constructor 种类

| 类型 | 说明 |
|------|------|
| **default constructor** | 无参数（或全部有默认值） |
| **conversion constructor** | 单参数，可做隐式类型转换 |
| **copy constructor** | `T(const T&)`，拷贝构造 |
| **general constructor** | 多参数自定义构造 |

- constructor 和类同名，无返回类型
- constructor 在以下时机被调用：
  1. 创建对象
  2. 按值传参（pass by value）
  3. 按值返回（return by value）

### 2.2 重要细节

- **compiler-generated default constructor**: 如果没写任何 constructor，编译器生成一个（什么都不做，成员值未初始化）
  - ⚠️ 一旦写了任何 constructor，compiler 就**不再**自动生成 default constructor
- **explicit** 关键字：阻止隐式转换
  ```cpp
  explicit Word(const char* s);
  // print_word("Titanic"); ← 这样就会报错
  ```
- **copy constructor** 被调用时机：
  1. 用已有对象初始化新对象 `Word b = a;` 或 `Word b(a);`
  2. pass by value
  3. return by value
- **copy elision / RVO**: 编译器优化，可能跳过 copy constructor

### 2.3 Default Memberwise Copy / Assignment 的问题 ⭐

- 如果不自定义 copy constructor / `operator=`，编译器做 **memberwise copy**（浅拷贝）
- 指针成员会导致 **两个对象共享同一块内存** → 析构时 double free
- ⭐ **Default copy assignment (`operator=`) 同理**：编译器生成的 `operator=` 也是 memberwise copy
  - 对于含指针成员的类，**必须同时自定义** copy constructor、copy assignment operator、destructor（**Rule of Three**）
  ```cpp
  Word& operator=(const Word& w) {
      if (this != &w) {          // 必须检查 self-assignment!
          delete [] str;          // 先释放旧资源
          str = new char[strlen(w.str)+1];
          strcpy(str, w.str);     // 深拷贝
          freq = w.freq;
      }
      return *this;
  }
  ```
  - ⭐ 如果不检查 self-assignment（`a = a`），会先 `delete` 自己的数据再读取，导致 **undefined behavior**

### 2.4 `{}` 统一初始化优势

```cpp
Word w {};      // default constructor（避免 Most Vexing Parse）
Word w1 {w2};   // copy constructor
```

### 2.5 Member Initializer List (MIL)

```cpp
Word(const char* s, int f = 1) : freq(f), lang('E') { ... }
```

- MIL 初始化 **发生在构造函数体之前**
- **const 成员和 reference 成员必须用 MIL**
- 成员对象也应用 MIL 选择其 constructor（否则会先 default construct 再赋值，效率低）
- 初始化顺序由**声明顺序**决定，不是 MIL 中的顺序

### 2.6 Delegating Constructor (C++11)

```cpp
Word(const Word& w) : Word(w.str, w.frequency) { cout << "copy\n"; }
```
- 一个 constructor 调用同类的另一个 constructor

### 2.7 Destructor

- `~T()` 无参数、无返回、只能有一个
- heap 内存必须手动 `delete`
- 析构顺序：**与构造顺序相反**（先构造的后析构）
- "has" 关系中：member 先构造，object 后构造；析构反过来

### 2.8 编译器自动生成的 6 个函数 (C++11)

1. default constructor
2. copy constructor
3. copy assignment operator (`operator=`)
4. move constructor
5. move assignment operator
6. destructor

- `= default`: 明确要求生成默认版本
- `= delete`: 禁止生成

## 3. Inheritance

### 3.1 基本概念

- **base class** (parent) → **derived class** (child)
- 语法: `class Derived : public Base { ... };`
- 意义: code reuse + logical hierarchy + extensibility + polymorphism

### 3.2 Access Control

| Base 成员 | public 继承 | protected 继承 | private 继承 |
|----------|-----------|--------------|-------------|
| public | public | protected | private |
| protected | protected | protected | private |
| private | 不可访问 | 不可访问 | 不可访问 |

- ⭐ **只有 public 继承是 is-a 关系**；protected/private 继承不会 slicing，会报错

### 3.3 构造/析构顺序

- 构造: **base → derived**（MIL 中调用 base constructor）
- 析构: **derived → base**

### 3.4 Function Overriding & Hiding

- derived 中同名同参函数 **override** base 版本
- 同名不同参 → **hiding**（base 版本被隐藏，需要 `Base::f()` 访问）

### 3.5 Virtual Function & Dynamic Binding

- `virtual` 函数 → **runtime polymorphism**（运行时多态）
- 通过 base pointer/reference 调用时，根据**实际类型**决定调用哪个版本
  ```cpp
  Animal* a = new Dog();
  a->speak(); // 调用 Dog::speak()（如果 speak 是 virtual）
  ```

### 3.6 Abstract Base Class (ABC) ⭐

- 含有 **pure virtual function** 的类: `virtual void foo() = 0;`
- ⭐ ABC 的规则：
  1. **不能定义对象**（不能实例化）
  2. derived class **必须实现** pure virtual function
  3. **不能 pass by value**，不能显式转换，但 **可以用 reference 和 pointer**
  4. **不能从 constructor 调用** pure virtual function
  5. 其他都可以有（非纯虚函数、数据成员等）；也可以有非 ABC 的 derived class

### 3.7 `final` 关键字 ⭐

```cpp
class Student final : public UPerson { /* ... */ };
```
- ⭐ 加了 `final` 的类**不可以被继承**

### 3.8 Multiple Inheritance & Diamond Problem

- 多继承：`class C : public A, public B { };`
- 同名冲突 → `A::f()` / `B::f()` 区分
- Diamond problem → **virtual inheritance** 解决
  ```cpp
  class B : virtual public A {};
  class C : virtual public A {};
  class D : public B, public C {};
  ```

### 3.9 Further Reading ⭐

- ⭐ **按值拷贝会按 `const&` 处理**（pass by value → const ref 语义）
- ⭐ protected/private 继承中 virtual function **只能在类内使用**

## 4. Generic Programming (Templates) & Operator Overloading

### 4.1 Function Template

```cpp
template <typename T>
inline const T& larger(const T& a, const T& b)
{ return (a < b) ? b : a; }
```

- ⭐ template 本身**不生成代码**；用到哪个类型才实例化哪个
- **template argument deduction**: 编译器从实参推断类型
- ⭐ **显式模板**: `func<typename>()`，当推断有歧义时使用
  ```cpp
  larger<double>(4, 5.5); // 不写<double>会报错
  ```
- ⭐ **指针和引用必须匹配类型**（template 不自动做类型转换）

### 4.2 Template Specialization ⭐

```cpp
template <>
const char* const& larger(const char* const& a, const char* const& b)
{ return (strcmp(a,b) < 0) ? b : a; }
```
- ⭐ 特化：加一个 `template<>` 前缀

### 4.3 Multiple Template Parameters ⭐

```cpp
template <typename T1, typename T2>
// 或
template <typename T, int a>  // non-type parameter
```

### 4.4 获取数组大小的技巧 ⭐

```cpp
template <typename T, int N>
int f(T (&x)[N]) { return N; }
```

### 4.5 Function Template vs Class Template ⭐

- ⭐ **function template 可以不写类型**（编译器推断）
- ⭐ **class template 必须写类型**: `List<int> primes;`

### 4.6 Template 放在 .h 文件 ⭐

- ⭐ template 一般写在 `.h` 文件，这样才能保证在同一个 translation unit 里，编译器才能生成对应函数

### 4.7 Template 的多种形态 ⭐

- ⭐ **Class Template 嵌套使用**（template of template）:
  ```cpp
  List<List<int>> nested_list;       // List 的元素也是 List
  List<List<List<char>>> deep_list;  // 多层嵌套
  ```
  - 编译器会为每一层分别实例化 template
  - 注意 `>>` 在 C++11 之前需要写成 `> >`（中间加空格）

- ⭐ **Template 与继承结合**:
  ```cpp
  template <typename T>
  class Base { /* ... */ };
  
  template <typename T>
  class Derived : public Base<T> { /* ... */ };
  ```

- ⭐ **Template 作为 template 参数**（template template parameter）:
  ```cpp
  template <template <typename> class Container, typename T>
  void print_all(const Container<T>& c);
  ```
  - 这允许把 container 类型本身作为参数传入

- ⭐ **Non-type parameter 与 type parameter 混合**:
  ```cpp
  template <typename T, int N>
  class FixedArray { T data[N]; };
  
  FixedArray<double, 100> arr;
  ```

### 4.8 Operator Overloading ⭐

- ⭐ `+` 和 `+=` **需要分别重载**
- ⭐ 不能重载的运算符: `. :: .* ?:`
- ⭐ **参数个数、结合律、优先级不能变**
- ⭐ `=`, `[]`, `()` **必须是 member function**
- ⭐ 如果操作左边是 class 对象，可以是 member function

#### `operator=` 与 Self-Assignment ⭐

- ⭐ 重载 `operator=` 时**必须检查 self-assignment** (`this != &rhs`)
  ```cpp
  T& operator=(const T& rhs) {
      if (this != &rhs) {
          // 释放旧资源 → 分配新资源 → 复制数据
      }
      return *this;
  }
  ```
- ⭐ 如果不检查：`a = a` 会先 delete 自己的数据，再尝试从已释放的内存复制 → **crash / undefined behavior**
- ⭐ Move assignment (`operator=(T&&)`) 同样需要检查：`if (this != &w)`

#### 前置/后置 ++ ⭐

```cpp
T& operator++();       // 前置 ++a
T  operator++(int);    // 后置 a++（int 是哑参数）
```
- ⭐ 必须这样写，前置返回引用，后置返回值

#### `operator<<` ⭐

- ⭐ `ostream&` 返回值所以可以链式输出
- ⭐ `operator<<` **不能是 member function**（因为 `cout` 在左边）
- ⭐ 通常声明为 **friend**:
  ```cpp
  friend ostream& operator<<(ostream& os, const T& a);
  ```

### 4.9 Friend

- `friend` 函数/类可以访问 private 成员
- friendship 不对称、不传递、不继承

## 5. Lambda & Function Pointer

### 5.1 Lambda 表达式

```
[ capture-list ] ( parameter-list ) mutable -> return-type { body }
```

- **capture list** 决定如何捕获外部变量：
  - `[=]`: 按值捕获所有局部变量
  - `[&]`: 按引用捕获所有局部变量
  - `[x, &y]`: x 按值，y 按引用
  - 全局变量**不需要也不能被捕获**

- **return type**: 无 return 则 void；有 return 自动推断

- **capture by value** 的时间点：lambda **定义时**复制（不是执行时）
- **capture by reference**: 直接操作原变量

- **mutable**: 允许修改按值捕获的变量副本
  ```cpp
  [b](int x) mutable { return b *= x; }(20); // OK
  ```

### 5.2 Function Pointer

```cpp
int (*fp)(int) = square;  // fp 是函数指针
```

- 函数指针可以动态选择函数
- 函数指针**不能指向有 capture 的 lambda**（无 capture 的 lambda 可以）

### 5.3 Nested Lambda

- lambda 内部可以再定义 lambda，capture 规则递归适用

## 6. BST (Binary Search Tree)

### 6.1 Big-O 复杂度回顾

| 函数 | 名称 |
|------|------|
| c | constant |
| log N | logarithmic |
| N | linear |
| N log N | linearithmic |
| N² | quadratic |
| 2^N | exponential |

### 6.2 Tree 基本概念

- **递归定义**: 空树，或 root + 若干 subtree
- 术语: root, parent, child, leaf, sibling, path, depth, height, ancestor, descendant

### 6.3 Binary Tree

- 每个节点**最多两个 children**
- 平均高度 O(log N)，最坏 O(N)

#### 三种 Traversal

| 遍历 | 顺序 |
|------|------|
| **inorder** | left → node → right |
| **preorder** | node → left → right |
| **postorder** | left → right → node |

- 对应 expression tree 的三种记法:
  - preorder → prefix notation
  - inorder → infix notation
  - postorder → postfix notation

### 6.4 BST 性质

- 左子树所有值 < 当前节点 < 右子树所有值
- **search / insert / delete** 平均 O(log N)，最坏 O(N)

### 6.5 BST 操作

- **search**: 递归比较，左或右走
- **find_min**: 一直往左走到底
- **find_max**: 一直往右走到底
- **insert**: 递归找到空位插入
- **delete** 三种情况：
  1. **叶节点**: 直接删
  2. **一个 child**: 用 child 替代
  3. **两个 children**: 用右子树最小值（或左子树最大值）替代，再递归删掉替代节点

## 7. Hashing

### 7.1 基本概念

- **hash table**: 固定大小的数组，用 hash function 将 key 映射到 slot
- 平均时间复杂度: **O(1)** (search / insert / delete)
- hash table 数据**无序**，不支持 find min/max、range query、sorted listing

### 7.2 Hash Function

- 最简单: `h(k) = k mod m`
- 选 m 的原则：
  - 避免 2 的幂
  - 避免 10 的幂
  - 选 **prime** 最好

#### String Hashing

1. 方法1: ASCII 值求和 → 排列(permutation)冲突，分布差
2. 方法2: `(key[0] + 27*key[1] + 27²*key[2]) mod m` → 好一些但前缀有限
3. 方法3: `Σ 37^(L-1-i) * key[i] mod m` → 用所有字符，分布最好

### 7.3 Collision Handling

#### 方法1: Separate Chaining

- 每个 slot 存一个 linked list
- 插入: 计算 `h(k)`，在链表头部插入
- 删除: 遍历链表找到并删除
- 优点: 删除简单
- 缺点: 链表额外内存 + allocation 慢

#### 方法2: Open Addressing

- 所有元素存在 table 内，冲突时重新探测
- probe 公式: `hi(k) = (hash(k) + f(i)) mod m`

| 策略 | f(i) | 特点 |
|------|------|------|
| **linear probing** | i | 简单但有 **primary clustering** |
| **quadratic probing** | i² | 减少 primary clustering；m 为素数且 ≤50% full 可保证插入 |
| **double hashing** | `i * hash2(k)` | 最好；`hash2(k) = R - (k mod R)`，R 为素数 |

- **primary clustering**: 连续占用的槽越来越长
- **secondary clustering**: 同一初始 hash 的 key 走相同探测路径

### 7.4 其他要点

- **lazy deletion** (open addressing): 标记 EMPTY / ACTIVE / DELETED，不能直接删
- **rehashing**: load factor `α = N/m` 过大时扩表，重新 hash 所有元素

## 8. rvalue Reference & Move Semantics

### 8.1 lvalue vs rvalue ⭐

- **lvalue**: 有地址，可以出现在赋值左边（如变量名）
- **rvalue (prvalue)**: 临时值，没有持久地址（如字面量、表达式结果）

### 8.2 Temporary Objects (临时对象) ⭐

- ⭐ `new` 出来的对象没名字，在 **heap** 上，必须用 pointer 管理，否则 memory leak
- ⭐ 临时对象在 **stack** 上，出现在以下情况：

  | 编号 | 情况 | 说明 |
  |------|------|------|
  | TO1 | const reference initialization | 绑定字面量/需要转换/表达式时产生临时对象 |
  | TO2 | argument passing | 字面量、转换、表达式作为 `T` 或 `const T&` 参数时 |
  | TO3 | function return by value | 返回值产生临时对象（除非 RVO） |
  | TO4 | expression evaluation | 子表达式求值 |

- ⭐ TO1 详细：绑定到左值 / 其他 ref / 函数返回的 ref → **无**临时对象；绑定字面量/需要转换/表达式 → **有**
- ⭐ TO2 详细：如果形参是 ref 或 ptr → 无；如果是字面量/转换/表达式且形参是 `T` 或 `const T&` → 有
- ⭐ TO3：只有 return by value 才有临时对象（除非 RVO 优化掉）

### 8.3 Temporary Object 的生命周期 ⭐

- 通常：持续到所在**完整表达式(full expression)结束**（分号处）
- ⭐ 如果被 rvalue ref 或 const ref 绑定 → **延长到引用变量的生命周期结束**
- ⭐ 但如果引用是函数参数 → 延长到**包含该函数调用的完整表达式完成**

### 8.4 rvalue Reference (`T&&`) ⭐

```cpp
int&& d = 5;        // OK: 绑定到 rvalue
int&& c = a;        // ERROR: 不能绑定到 lvalue
```

- ⭐ 和左值引用一样：**必须初始化，不能改 binding**
- ⭐ 和左值引用的区别：**不能绑 lvalue，只能绑临时对象**
- ⭐ `std::move` **不影响生命周期**！只是类型转换，不会真的变成临时变量

| 特性 | `const T&` | `T&&` |
|------|-----------|-------|
| 可修改 | ✘ | ✔ |
| 接受 lvalue | ✔ | ✘ |
| 接受 rvalue | ✔ | ✔ |

- 函数重载解析：如果同时有 `f(const T&)` 和 `f(T&&)`，临时对象优先匹配 `T&&`

### 8.5 Move Semantics ⭐

- 核心思想：临时对象即将被销毁，可以**窃取其资源**而非复制

#### Move Constructor

```cpp
Word(Word&& w) : freq(w.freq), str(w.str) {
    w.freq = 0;
    w.str = nullptr;  // 转移所有权，原对象置空
}
```

#### Move Assignment

```cpp
Word& operator=(Word&& w) {
    if (this != &w) {
        delete [] str;
        freq = w.freq;
        str = w.str;
        w.freq = 0;
        w.str = nullptr;
    }
    return *this;
}
```

### 8.6 `std::move()`

```cpp
std::move(object)  // 将 lvalue 强制转换为 rvalue reference
```

- ⭐ `std::move()` **本身不做任何移动**，只是类型转换（cast）
- 用途：强制调用 move constructor / move assignment

### 8.7 Return 时的规则 ⭐

- return by value → 调用 copy/move constructor（可能被 RVO 优化）
- return 局部变量 → 编译器可能自动使用 move（即使没写 `std::move`）

## Self-Study 1: Separate Compilation & Makefile

### SS1.1 分离编译

- `.cpp` → `g++ -c` → `.o`（object file）
- 多个 `.o` → `g++ -o` → executable（链接）

### SS1.2 Dependencies

- 如果 `.h` 改了，所有 include 它的 `.cpp` 都要重新编译
- 手动跟踪依赖很麻烦 → 用 **Makefile**

### SS1.3 Makefile

```make
TARGET: DEPENDENCIES
    COMMAND
```

- `g++ -MM *.cpp` 自动生成依赖关系
- 变量: `SRCS`, `OBJS`, `CXXFLAGS`
- `$@` = target, `$<` = 第一个依赖

### SS1.4 Static vs Dynamic Linking

| 类型 | 特点 |
|------|------|
| static | 库代码复制进可执行文件，更快但更大 |
| dynamic | 运行时引用共享库，更小但略慢 |

### SS1.5 Header Guard

```cpp
#ifndef LAMP_H
#define LAMP_H
// ...
#endif
```
- 防止头文件被重复包含

## Self-Study 2: Namespace

### SS2.1 问题

- 两个库有同名 class / function → 编译冲突

### SS2.2 解决: namespace

```cpp
namespace apple { class Stack { ... }; }
namespace microsoft { class Stack { ... }; }
```

- 访问: `apple::Stack`, `microsoft::Stack`
- 别名: `namespace ms = microsoft;`

### SS2.3 using 声明

```cpp
using apple::Some_Class;   // 只引入特定名字
using namespace apple;     // 引入全部（可能冲突，不推荐）
```

### SS2.4 `std` namespace

- 标准库都在 `namespace std` 中
- `using namespace std;` 虽然方便但全局使用**不推荐**

### SS2.5 namespace 特性

- 可以分多次定义（expandable）
- 可以嵌套: `hkust::cse::rank()`

## Self-Study 3: Static Data Members & Member Functions

### SS3.1 static 变量

- **file scope**: `static int x;` — 只在当前文件可见
- **function scope**: 函数内的 `static` 变量只初始化一次，跨调用保持值

### SS3.2 Static Class Data Members

- 所有对象**共享同一份**
- 即使没有对象也存在
- **不存在对象内部**
- 必须在 `.cpp` 文件中**类外定义**
  ```cpp
  int Car::num_cars = 0;
  ```

### SS3.3 Static Member Functions

- 属于**类**而非对象
- 调用方法: `ClassName::function()` 或 `object.function()`
- **没有 `this` 指针**
- 只能访问 static data members
- 不能是 `const`，不能是 `virtual`

## Self-Study 4: STL (Standard Template Library)

### SS4.1 三大组件

```
STL = Containers + Iterators + Algorithms
```

### SS4.2 Container 分类

| 类别 | 容器 |
|------|------|
| Sequence | `vector`, `list`, `deque` |
| Associative | `map`, `multimap`, `set`, `multiset` |
| Adaptors | `stack`, `queue`, `priority_queue` |
| Near-containers | `bitset`, `valarray`, `string` |

### SS4.3 Sequence Container

| 容器 | random access | insert/delete |
|------|-------------|---------------|
| `vector` | O(1) | O(1) at end |
| `list` | O(n) | O(1) anywhere |
| `deque` | O(1) | O(1) front/back |

常用操作: `push_back()`, `pop_back()`, `front()`, `back()`, `operator[]`, `size()`, `empty()`

### SS4.4 Container Adaptors

- **stack** (LIFO): `top()`, `push()`, `pop()`
- **queue** (FIFO): `front()`, `back()`, `push()`, `pop()`

### SS4.5 Iterator

- 泛化的指针，用于遍历容器
- 操作: `*p`, `++p`, `--p`, `p == q`
- `begin()` / `end()`

```cpp
for (list<int>::const_iterator p = x.begin(); p != x.end(); ++p)
    cout << *p;
```

### SS4.6 STL Algorithms

- `find`, `find_if`, `count_if`, `for_each`, `transform`, `sort`, `binary_search`, `merge`, `unique`, `reverse`, `remove_if` 等
- 通过 **iterator** 操作容器，与容器类型解耦

### SS4.7 Function Object (Functor)

- 重载 `operator()` 的对象，行为像函数
```cpp
class Greater_Than {
    int limit;
public:
    Greater_Than(int a) : limit(a) {}
    bool operator()(int value) { return value > limit; }
};
find_if(x.begin(), x.end(), Greater_Than(350));
```
