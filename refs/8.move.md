# lvalue, rvalue & xvalue of a Variable

A **variable** is a symbolic name assigned to some memory storage.

Difference between a **variable** and a **literal constant**:

- A variable **has an address**
- A literal constant **does not**

Example:

```cpp
x = 100;
```

- `x` → variable  
- `100` → literal constant  

A variable has **two roles depending on context**.

Example:

```cpp
x = x + 1;
```

- **lvalue** → its location (read-write)
- **prvalue (pure rvalue)** → its value (read-only)

Example code:

```cpp
int x; // OK
4 = 1; // Error
(x + 10) = 6; // Error
```

# Unnamed Objects I: Dynamically Allocated Objects

Syntax:

```cpp
T* <variable> = <dynamic object>;
```

Examples:

```cpp
int* ip = new int;
Word* wp = new Word;
Student* sp = new Student;
```

Dynamic objects created by `new`:

- have **no name**
- must be stored using **pointers**

Memory location:

```
heap
```

If all pointers are lost → **memory leak**.

# Unnamed Objects II: Temporary Objects

Temporary objects are created automatically on the **stack**.

Situations where they appear:

```
TO1 const reference initialization
TO2 argument passing
TO3 function return value
TO4 expression evaluation
```

Example:

```
result of sub-expressions
type conversions
```

Temporary objects are:

- created automatically
- destroyed automatically

---

# Before C++11: const T&

Syntax:

```cpp
const T& <variable> = <temporary object>;
```

Purpose:

- extend the lifetime of a temporary object

Limitation:

- cannot modify the object

Reason:

```
temporary objects are rvalues
```

# Temporary Values Example (Basic Types)

```cpp
#include <iostream>
using namespace std;

int square(int x) { return x*x; }

void cbv(int x) { cout << "call-by-value: " << x << endl; }
void cbr(int& x) { cout << "call-by-ref: " << x << endl; }
void cbcr(const int& x) { cout << "call-by-const-ref: " << x << endl; }

int main()
{
    int a = 3;

    int& b = 4; // Error
    const int& c = 5; // temporary object

    int d = square(3); // returned temporary
    int e = a + c + d; // expression temporary

    cbv(a); // OK
    cbr(a); // OK
    cbr(8); // Error

    cbcr(8); // OK

    return 0;
}
```

Important rule:

```
lvalue reference → binds only to lvalue
const lvalue reference → can bind to rvalue
```

# Class Word Example

```cpp
#include <iostream>
#include <cstring>
using namespace std;

class Word
{
private:
    int freq = 0;
    char* str = nullptr;

public:

    Word() { cout << "default constructor" << endl; }

    Word(const char* s, int f = 1)
        : freq(f), str(new char[strlen(s)+1])
    {
        strcpy(str, s);
        cout << "conversion: ";
        print();
    }

    Word(const Word& w)
        : freq(w.freq+1), str(new char[strlen(w.str)+1])
    {
        strcpy(str, w.str);
        cout << "copy: ";
        print();
    }

    ~Word()
    {
        cout << "destructor: ";
        print();
        delete [] str;
    }

    void print() const
    {
        cout << (str ? str : "null") << " ; " << freq << endl;
    }
};
```

# Lifetime of Temporary Objects

Rules:

Temporary objects are destroyed:

```
at end of expression
```

Exceptions:

If bound to

```
const reference
rvalue reference
```

they survive until the reference goes out of scope.

# After C++11: rvalue Reference

Syntax:

```cpp
T&& variable = temporary_object;
```

Definition:

```
rvalue reference = alias to temporary object
```

Purpose:

- allow manipulation of temporary objects
- improve efficiency

Important properties:

- must be initialized
- cannot bind to lvalue
- behaves like a normal variable once created

---

# rvalue Reference Example (int)

```cpp
#include <iostream>
using namespace std;

int square(int x) { return x*x; }

int main()
{
    int a = 8;

    int&& b; // Error
    int&& c = a; // Error

    int&& d = 5;
    int&& e = square(5);

    cout << d << endl;
    cout << e << endl;

    d = e = 10;

    cout << d << '\t' << e << endl;

    return 0;
}
```

---

# Why rvalue Reference Exists

Originally:

```
references only bind to lvalues
```

Temporary objects:

```
treated as rvalues
```

But sometimes we want to:

- modify them
- reuse their memory resources

Thus:

```
rvalue references were introduced
```

---

# const lvalue Reference vs rvalue Reference

Similarities:

- both bind to temporary objects
- both must be initialized

Differences:

| Feature | const T& | T&& |
|-------|-------|-------|
| Modifiable | No | Yes |
| Accepts lvalue | Yes | No |
| Accepts rvalue | Yes | Yes |

Function resolution rule:

If both exist:

```
f(const T&)
f(T&&)
```

and argument is temporary → **T&& chosen**

# Move Semantics Idea

Observation:

```
temporary objects will be destroyed soon
```

Thus we can:

```
steal their resources
```

Instead of copying memory.

Example improvement:

```
copy constructor → move constructor
copy assignment → move assignment
```

Benefits:

```
no memory allocation
better performance
```

---

# Move Constructor Example

```cpp
Word(Word&& w)
    : freq(w.freq), str(w.str)
{
    w.freq = 0;
    w.str = nullptr;

    cout << "move: ";
    print();
}
```

Key idea:

```
transfer ownership of resources
```

---

# Move Assignment Operator

```cpp
Word& operator=(Word&& w)
{
    if (this != &w)
    {
        delete [] str;

        freq = w.freq;
        str = w.str;

        w.freq = 0;
        w.str = nullptr;

        cout << "move assignment: ";
        print();
    }

    return *this;
}
```

---

# std::move()

Syntax:

```cpp
std::move(object)
```

Meaning:

```
cast object to rvalue reference
```

Important fact:

```
std::move() does NOT move anything
```

It only **casts the type**.

---

# std::move Example

```cpp
Word_Pair(Word& a, Word& b)
    : w1(std::move(a)), w2(std::move(b))
{
}
```

Effect:

```
force move constructor
instead of copy
```

# Compiler-generated Member Functions

If not defined, the compiler may generate:

1. default constructor  
2. copy constructor  
3. copy assignment operator  
4. move constructor (C++11)  
5. move assignment operator (C++11)  
6. destructor  

C++11 allows explicit control:

Generate:

```cpp
= default;
```

Disable:

```cpp
= delete;
```

# rvalue Reference Example with string

```cpp
#include <iostream>
using namespace std;

string wrap(string s)
{
    return "begin." + s + ".end";
}

int main()
{
    string s1 {"w"};

    string&& s2; // error
    string&& s3 = s1; // error

    string&& s4 = "x";
    cout << s4 << endl;

    string&& s5 = wrap("x");
    cout << s5 << endl;

    s4 = "z";
    cout << s4 << endl;

    s5 = s1;
    cout << s5 << endl;

    return 0;
}
```

# Key Takeaways

- **lvalue** → object with memory location  
- **rvalue** → temporary value  
- **rvalue reference (`T&&`)** → reference to temporary object  

Move semantics allows:

```
transfer resources instead of copying
```

Major benefits:

```
higher efficiency
less memory allocation
better performance
```
