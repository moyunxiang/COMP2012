# Inheritance

Inheritance allows a class to **reuse, extend, and modify** the behavior of another class.

Terminology:

- **Base class (parent class)**
- **Derived class (child class)**

Basic syntax:

```cpp
class Derived : access_specifier Base
{
};
```

Example:

```cpp
class Animal
{
public:
    void eat() { cout << "Animal eats\n"; }
};

class Dog : public Animal
{
public:
    void bark() { cout << "Dog barks\n"; }
};
```

---

# Motivation of Inheritance

Inheritance supports:

- **Code reuse**
- **Logical hierarchy**
- **Extensibility**

Example hierarchy:

```
Animal
 ├── Dog
 ├── Cat
 └── Bird
```

Each derived class inherits the properties of `Animal`.

# Base Class and Derived Class

Example:

```cpp
class Person
{
protected:
    string name;

public:
    Person(string n) : name(n) {}
    void printName() { cout << name << endl; }
};

class Student : public Person
{
private:
    int studentID;

public:
    Student(string n, int id) : Person(n), studentID(id) {}
};
```

# Access Control in Inheritance

Three inheritance modes:

```
public inheritance
protected inheritance
private inheritance
```

Effect on member access:

| Base Member | public inheritance | protected inheritance | private inheritance |
|--------------|------------------|----------------------|--------------------|
| public | public | protected | private |
| protected | protected | protected | private |
| private | not accessible | not accessible | not accessible |

---

# Public Inheritance

Most common form.

Meaning:

```
Derived IS-A Base
```

Example:

```cpp
class Vehicle
{
public:
    void move() {}
};

class Car : public Vehicle
{
};
```

A **Car IS-A Vehicle**.

---

# Protected and Private Inheritance

Protected inheritance:

- public → protected
- protected → protected

Private inheritance:

- public → private
- protected → private

Example:

```cpp
class A
{
public:
    int x;
protected:
    int y;
};

class B : protected A
{
};
```

# Constructor and Inheritance

Base class constructor is called **before** derived class constructor.

Example:

```cpp
class Base
{
public:
    Base() { cout << "Base constructor\n"; }
};

class Derived : public Base
{
public:
    Derived() { cout << "Derived constructor\n"; }
};
```

Output:

```
Base constructor
Derived constructor
```

# Using Member Initializer List for Base Classes

Derived constructors must initialize the base class:

```cpp
class Person
{
protected:
    string name;

public:
    Person(string n) : name(n) {}
};

class Student : public Person
{
private:
    int id;

public:
    Student(string n, int i) : Person(n), id(i) {}
};
```

# Destructor Order

Destruction happens in **reverse order**.

Example:

```cpp
class Base
{
public:
    ~Base() { cout << "Base destructor\n"; }
};

class Derived : public Base
{
public:
    ~Derived() { cout << "Derived destructor\n"; }
};
```

Output:

```
Derived destructor
Base destructor
```

# Overriding Functions

Derived classes can override base class functions.

Example:

```cpp
class Animal
{
public:
    void speak() { cout << "Animal sound\n"; }
};

class Dog : public Animal
{
public:
    void speak() { cout << "Bark\n"; }
};
```

---

# Function Hiding

If derived class defines a function with the same name, it hides the base class version.

Example:

```cpp
class Base
{
public:
    void f() { cout << "Base f\n"; }
};

class Derived : public Base
{
public:
    void f(int x) { cout << "Derived f\n"; }
};
```

`Base::f()` becomes hidden.

Use scope resolution to access it:

```cpp
Base::f();
```

---

# Calling Base Class Functions

Inside derived class:

```cpp
Base::functionName();
```

Example:

```cpp
class Derived : public Base
{
public:
    void show()
    {
        Base::show();
    }
};
```

# Multiple Inheritance

A class may inherit from **multiple base classes**.

Example:

```cpp
class Printer
{
public:
    void print() {}
};

class Scanner
{
public:
    void scan() {}
};

class Copier : public Printer, public Scanner
{
};
```

---

# Ambiguity in Multiple Inheritance

If two base classes define the same function:

```cpp
class A
{
public:
    void f() {}
};

class B
{
public:
    void f() {}
};

class C : public A, public B
{
};
```

Calling `f()` becomes ambiguous.

Solution:

```cpp
A::f();
B::f();
```

---

# Diamond Problem

Example:

```
        A
       / \
      B   C
       \ /
        D
```

D inherits A twice.

Solution: **virtual inheritance**

```cpp
class A {};

class B : virtual public A {};
class C : virtual public A {};

class D : public B, public C {};
```

---

# Virtual Functions

Used for **runtime polymorphism**.

Example:

```cpp
class Animal
{
public:
    virtual void speak()
    {
        cout << "Animal sound\n";
    }
};

class Dog : public Animal
{
public:
    void speak() override
    {
        cout << "Bark\n";
    }
};
```

---

# Dynamic Binding

Example:

```cpp
Animal* a = new Dog();
a->speak();
```

Output:

```
Bark
```

This happens because of **virtual functions**.

---

# Abstract Classes

A class containing **pure virtual functions**.

Example:

```cpp
class Shape
{
public:
    virtual double area() = 0;
};
```

Derived classes must implement it.

---

# Pure Virtual Function

Syntax:

```cpp
virtual returnType function() = 0;
```

Example:

```cpp
class Shape
{
public:
    virtual double area() = 0;
};

class Circle : public Shape
{
public:
    double area() override
    {
        return 3.14;
    }
};
```

# Interface Concept

Abstract classes often act as **interfaces**.

Example:

```cpp
class Drawable
{
public:
    virtual void draw() = 0;
};
```
