# Class Object Initialization

If all data members of a class are **public** (so the class is actually a basic struct), they can be initialized when they are created using the brace initializer `{}`.

```cpp
class Word /* File: public-member-init.cpp */
{
public:
    int frequency;
    const char* str;
};

int main() { Word movie = {1, "Titanic"}; }
```

---

## What happens if some members are private?

```cpp
class Word /* File: private-member-init.cpp */
{
public:
    int frequency;
private:
    const char* str;
};

int main() { Word movie = {1, "Titanic"}; }
```

Error:

```
error: could not convert {1, "Titanic"} from <brace-enclosed initializer list> to Word
```

# Part I  
# Constructors

---

## Different Types of C++ Constructors

Conceptual analogy:

- **Default constructor** → blank iPhone
- **Conversion constructor** → Android phone to iPhone migration
- **Copy constructor** → migrate an old iPhone to a new iPhone
- **Other constructors** → customized iPhone

---

## Constructor Member Functions

```cpp
Word movie;                    // Default constructor
Word director = "J. Cameron";  // Implicit conversion constructor
Word sci_fi("Avatar");         // Explicit conversion constructor
Word drama {"Titanic"};        // C++11 explicit conversion constructor
Word* p = new Word("action",1);// General constructor
```

Properties of constructors:

- A constructor has the **same name as the class**
- A constructor **has no return type**
- A constructor **cannot explicitly return a value**

A constructor is called when:

- an object is created
- an object is passed to a function **by value**
- an object is **returned by value**

---

## Default Initializers for Non-static Data Members (C++11)

```cpp
class Word /* File: default-initializer.cpp */
{
    int frequency {0};
    const char* str {nullptr};
};

int main() { Word movie; }
```

If the constructor does not initialize a data member:

- it uses the **default initializer** if one exists
- otherwise the value is **undefined**

---

## Default Constructor

```cpp
class Word /* File: default-constructor.cpp */
{
private:
    int frequency;
    char* str;

public:
    Word() { frequency = 0; str = nullptr; }
};

int main()
{
    Word movie;
}
```

Purpose:

- create objects with **default values**

---

## Conversion Constructors

```cpp
class Word
{
private:
    int frequency;
    char* str;

public:
    Word(char c)
    {
        frequency = 1;
        str = new char[2];
        str[0] = c;
        str[1] = '\0';
    }

    Word(const char* s)
    {
        frequency = 1;
        str = new char[strlen(s)+1];
        strcpy(str,s);
    }
};
```

Example:

```cpp
Word movie("Titanic");
Word movie2 {'A'};
Word movie3 = 'B';
```

# Copy Constructor

```cpp
class Word
{
private:
    int frequency;
    char* str;

public:
    Word(const char* s, int k = 1)
    {
        frequency = k;
        str = new char[strlen(s)+1];
        strcpy(str,s);
        cout << "conversion\n";
    }

    Word(const Word& w)
    {
        frequency = w.frequency;
        str = new char[strlen(w.str)+1];
        strcpy(str,w.str);
        cout << "copy\n";
    }
};
```

Example:

```cpp
Word movie("Titanic");
Word song(movie);
Word ship = movie;
Word actress {"Kate"};
```

---

## When Copy Constructor is Called

Occurs when:

- passing objects **by value**
- initializing objects using `=`
- returning objects **by value**

---

## Return-by-Value Example

```cpp
Word to_upper_case() const
{
    Word x(*this);

    for(char* p = x.str; *p!='\0'; p++)
        *p += 'A' - 'a';

    return x;
}
```

---

## Copy Elision / Return Value Optimization

Compiler optimization:

- avoids unnecessary copy constructor calls
- constructs the return value **directly**

---

# Default Copy Constructor

If none defined:

```
X(const X&) { memberwise copy }
```

Example:

- copy each member
- copy arrays element-by-element

---

# Default Memberwise Assignment

If assignment operator `=` not defined:

```
song.frequency = movie.frequency
song.str = movie.str
```

This causes problems with **pointer members**.

---

## Memberwise Assignment Problem

Pointer copying causes **two objects sharing the same pointer**.

Example:

```
x.str -> "rat"
z.str -> "cat"

z = x

Both now point to "rat"
```

Memory management problems occur.

# Part II  
# Member Initializer List (MIL)

---

## Member Initializer List

Initialization before constructor body:

```
: member1(expr1), member2(expr2)
```

Example:

```cpp
class Word
{
private:
    char lang;
    int freq;
    char* str;

public:
    Word() : lang('E'), freq(0), str(nullptr) {}

    Word(const char* s, int f = 1, char g = 'E')
        : lang(g), freq(f)
    {
        str = new char[strlen(s)+1];
        strcpy(str,s);
    }
};
```

---

## Initialization of Member Objects

```cpp
class Word_Pair
{
private:
    Word w1;
    Word w2;

public:
    Word_Pair(const char* s1, const char* s2)
        : w1(s1,5), w2(s2) {}
};
```

---

## Why MIL Is Important

Without MIL:

```
w1 = x;
w2 = y;
```

This causes **assignment**, not construction.

---

## const and Reference Members

Must be initialized using MIL.

```cpp
class Example
{
    const int const_m;
    int& ref_m;

public:
    Example(int c, int& r)
        : const_m(c), ref_m(r) {}
};
```

---

# Delegating Constructor (C++11)

```cpp
class Word
{
private:
    int frequency;
    char* str;

public:
    Word(const char* s, int f = 1)
    {
        frequency = f;
        str = new char[strlen(s)+1];
        strcpy(str,s);
    }

    Word(const Word& w)
        : Word(w.str, w.frequency)
    {
        cout << "copy\n";
    }
};
```

# Part III  
# Garbage Collection & Destructor

---

## Memory Layout

Program memory consists of:

- program code
- static/global data
- stack
- heap

Example:

```cpp
int* p = new int[100];
```

- pointer on stack
- array on heap

---

## Runtime Stack vs Heap

- stack variables created automatically
- destroyed when function ends
- heap memory persists until **delete**

Memory leak:

```
allocated memory without references
```

---

# Destructor

Destructor syntax:

```
~X()
```

Properties:

- no arguments
- no return type
- only one destructor per class

Default destructor:

```cpp
X::~X() {}
```

---

## User-defined Destructor

```cpp
class Word
{
private:
    int frequency;
    char* str;

public:
    Word(const char* s)
    {
        str = new char[strlen(s)+1];
        strcpy(str,s);
    }

    ~Word()
    {
        delete [] str;
    }
};
```

---

## Deleting Objects

```cpp
delete p;
delete [] x;
```

# Part IV  
# Order of Construction & Destruction

---

## "Has" Relationship

```cpp
class B {};

class A
{
private:
    B my_b;
};
```

A **has** B.

---

## Construction Example

```cpp
class Clock
{
public:
    Clock(){ cout<<"Clock Constructor\n"; }
    ~Clock(){ cout<<"Clock Destructor\n"; }
};

class Postoffice
{
    Clock clock;

public:
    Postoffice(){ cout<<"Postoffice Constructor\n"; }
    ~Postoffice(){ cout<<"Postoffice Destructor\n"; }
};
```

Output:

```
Clock Constructor
Postoffice Constructor
Postoffice Destructor
Clock Destructor
```

---

## Ownership Relationship

If class owns pointer:

```cpp
Clock* clock;
```

Destructor must delete it:

```cpp
~Postoffice()
{
    delete clock;
}
```

---

## Order Rules

1. Data members constructed **before** object constructor body
2. Destruction happens in **reverse order**
3. Order determined by **declaration order**

---

# Array Construction & Destruction

```cpp
Word weather[] { "sunny", "cloudy", "rainy" };
```

Constructors run in order:

```
sunny
cloudy
rainy
```

Destructors run in reverse.

---

# Final Summary

When an object is constructed:

- its data members are constructed first

When destructed:

- member destructors run **after object destructor code**

When objects own heap memory:

- destructors must release memory

Member initializer list allows:

- selecting constructors for member objects
- efficient initialization
