# Generic Programming Using Templates

Often functions or data structures differ only in the **types of objects they manipulate**.

Instead of duplicating code, we can use:

- **Function templates**
- **Class templates**

Generic programming allows writing **one version of code** that works for multiple types.

---

# Function Template of `larger()`

```cpp
template <typename T>
inline const T& larger(const T& a, const T& b)
{
    return (a < b) ? b : a;
}
```

`typename` can also be written as `class`.

```cpp
template <class T>
inline const T& larger(const T& a, const T& b)
{
    return (a < b) ? b : a;
}
```

Note:  
A template itself **is not a real function**.  
The compiler generates functions when needed.

---

# Using the Template

```cpp
#include <iostream>
using namespace std;

template <typename T>
inline const T& larger(const T& a, const T& b)
{
    return (a < b) ? b : a;
}

int main()
{
    int x = 4, y = 8;
    cout << larger(x, y) << endl;

    string a("cheetah"), b("gorilla");
    cout << larger(a, b) << endl;

    return 0;
}
```

---

# Template Instantiation

The compiler generates functions based on usage.

Example instantiations:

```
larger<int>()
larger<string>()
```

Each type generates **a separate compiled function**.

---

# Template Argument Deduction

```cpp
cout << larger(3, 5);      // T = int
cout << larger(4.3, 5.6);  // T = double
```

Compiler deduces template parameters from function arguments.

---

# Explicit Template Instantiation

Templates **do not automatically perform type conversion**.

Example:

```cpp
cout << larger(4, 5.5); 
```

This causes a compile error.

Solution:

```cpp
cout << larger<double>(4, 5.5);
```

---

# Template Problem Example

```cpp
template <typename T>
const T& larger(const T& a, const T& b)
{
    return (a < b) ? b : a;
}

int main()
{
    const char* m = "microsoft";
    const char* a = "apple";

    cout << larger(a, m) << endl;
}
```

Problem:

`<` compares **addresses**, not string content.

---

# Template Specialization

```cpp
template <>
const char* const& larger(const char* const& a, const char* const& b)
{
    return (strcmp(a, b) < 0) ? b : a;
}
```

---

# Template Specialization Example (Student)

```cpp
template <>
const Student& larger(const Student& a, const Student& b)
{
    return (a.get_GPA() < b.get_GPA()) ? b : a;
}
```

# Templates With Multiple Type Parameters

```cpp
template <typename T1, typename T2>
inline const T1& larger(const T1& a, const T2& b)
{
    return (a < b) ? b : a;
}
```

Usage:

```cpp
larger(4, 5.5);
larger(5.5, 4);
```

---

# Problem: Returning References

Returning `T1&` while returning a `T2` object can cause problems.

Solution 1:

```cpp
template <typename T1, typename T2>
inline T1 larger(const T1& a, const T2& b)
{
    return (a < b) ? b : a;
}
```

---

# Alternative Solution

```cpp
template <typename T1, typename T2>
void print_larger(const T1& a, const T2& b)
{
    if (a > b)
        cout << a;
    else
        cout << b;
}
```

---

# Template Code Bloat

Example:

```
print_larger(s, s)
print_larger(s, c)
print_larger(c, s)
...
```

Each combination creates **a separate compiled function**.

This may increase executable size.

# Class Template Example

```cpp
template <typename T>
class List_Node
{
public:
    List_Node(const T& x) : data(x) {}

    T data;
    List_Node* next {nullptr};
};
```

---

# Template List Class

```cpp
template <typename T>
class List
{
public:
    void prepend(const T& item)
    {
        List_Node<T>* new_node = new List_Node<T>(item);

        if (!head)
            head = new_node;
        else {
            new_node->next = head;
            head = new_node;
        }
    }

    void print() const
    {
        for (const List_Node<T>* p = head; p; p = p->next)
            cout << p->data << endl;
    }

private:
    List_Node<T>* head {nullptr};
};
```

---

# Using the Template List

```cpp
List<char> letters;
letters.prepend('a');
letters.prepend('b');

List<int> primes;
primes.prepend(2);
primes.prepend(3);
```

# Vector Example

```cpp
class Vector
{
public:
    Vector(double a = 0, double b = 0) : x(a), y(b) {}

    double getx() const { return x; }
    double gety() const { return y; }

private:
    double x, y;
};
```

---

# Global Operator+

```cpp
Vector operator+(const Vector& a, const Vector& b)
{
    return Vector(a.getx()+b.getx(), a.gety()+b.gety());
}
```

Usage:

```
d = a + b + c;
```

---

# Operator Function Syntax

Formal call:

```cpp
operator+(a,b)
```

Normal syntax:

```
a + b
```

Operators are **syntactic sugar for functions**.

# Part IV  
## Further Reading

Templates with **multiple parameters and different return types**.

Example issue:

```cpp
template <typename T1, typename T2>
inline const T1& larger(const T1& a, const T2& b)
{
    return (a < b) ? b : a;
}
```

Returning references to temporary objects can cause **runtime errors**.

Solution:

```cpp
template <typename T1, typename T2>
inline T1 larger(const T1& a, const T2& b)
{
    return (a < b) ? b : a;
}
```

Return **by value instead of reference**.
